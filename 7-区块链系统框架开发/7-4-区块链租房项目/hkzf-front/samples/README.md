# 测试数据说明

业务模型与完整流程见项目根目录 **[docs/业务模型.md](../../docs/业务模型.md)**。

## 导入顺序

1. 管理端上传 `auth.csv`（个人认证）
2. 等待约 5 秒写入链上
3. 管理端上传 `house.csv`（房产认证）
4. 等待约 5 秒
5. 用户端首页发起租赁申请，按三步走完流程

## CSV 格式（无表头，每行 3 列）

### auth.csv
```
211004197001010000,张三,false
211004199003030000,王五,true
```

### house.csv
```
1111112222223333333000000000,211004197001010000,true
```

## 前台测试用例

| 步骤 | 填写内容 |
|------|----------|
| 身份认证 | 姓名 `张三`，身份证 `211004197001010000` |
| 房产认证 | 房产证 `1111112222223333333000000000`，身份证 `211004197001010000` |
| 签约存证 | 上传 `HT2026001.pdf`，合同编号 `HT2026001` |

## 反面用例

- 王五（`211004199003030000`）有不良记录，身份认证应拦截
- 跳过前两步直接打开签约页，会被重定向
- 未导入的数据查询返回 `false:false`
