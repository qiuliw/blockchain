# 测试数据说明

## 导入顺序

1. 上传 `auth.csv`（个人认证）
2. 等待约 5 秒写入链上
3. 上传 `house.csv`（房产认证）
4. 等待约 5 秒
5. 在前台按示例信息走认证流程，并在签约存证页上传 `HT2026001.pdf`

## CSV 格式（无表头，每行 3 列）

### auth.csv
```
身份证号,姓名,是否有不良记录
211004197001010000,张三,false
```

### house.csv
```
房产证号,身份证号,是否可出租
1111112222223333333000000000,211004197001010000,true
```

## 前台测试用例（导入 auth + house 后）

| 步骤 | 填写内容 |
|------|----------|
| 身份认证 | 姓名 `张三`，身份证 `211004197001010000` |
| 房产认证 | 房产证 `1111112222223333333000000000`，身份证 `211004197001010000` |
| 签约存证 | 上传 `HT2026001.pdf`，合同编号自动为 `HT2026001` |

## 反面用例

- 王五（`211004199003030000`）有不良记录，身份认证应提示无法继续
- 未导入的数据查询会返回 `false:false`
